# Project Memory

Project memory stores durable creative context:

- platform,
- audience,
- creator persona,
- style preferences,
- previous outputs,
- feedback,
- asset history,
- successful Skill-Plus package variants.

Memory should be used as compile-time context for Skill-Plus packages and as review context for generated artifacts.

The broader direction is [Creative Continuity Architecture](creative-continuity.md):
project memory should evolve into model-readable creative spaces with
canon, decisions, episodes, assets, feedback, retrieval, ranking, and
long-term planning.
